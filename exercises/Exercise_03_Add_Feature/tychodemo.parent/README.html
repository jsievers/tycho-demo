Exercise 3: Add a feature

In this exercise, we create a feature which groups the plugin(s) we build to units which should be installed together. A feature is needed in order to include content into a p2 repository.

Recovery option

This section is optional and only needed if you want to recover from a problem in exercise 2.
  • Select File > Switch Workspace > Other... and choose the folder Exercise_03_Add_Feature
  • Import all three projects from this folder into the workspace using File > Import > Existing Maven Projects
  • Trigger a build on the parent POM:
    • Right-click on tychodemo.parent > Run As > Maven build
    • If a launch configuration dialog appears, enter the goals clean install and hit Run
  • The build should succeed and build three modules 
       tychodemo.parent 
   tychodemo.bundle 
   tychodemo.bundle.tests

Adding a feature

  1. Create a new feature project tychodemo.feature and add tychodemo.bundle as included plugin:
    File > New > Other > Plug-in Development > Feature Project



    Note: Make sure the new project root folder is located next to the existing module folders (Uncheck "Use default location" in the project creation wizard and enter location if necessary)
  2. Same as in exercise 1, convert the project tychodemo.feature to a Maven project.
    Enter groupId: tychodemo, Version: 1.0.0-SNAPSHOT, Packaging: eclipse-feature:

    Note packaging type eclipse-feature. Also note POM version == feature version and POM artifactId == feature id.
  3. Same as in exercise 2, add the new module tychodemo.feature to the parent pom tychodemo.parent.
  4. tychodemo.parent > Run As > Maven build
    Expected result: SUCCESSFUL build with 4 modules, including tychodemo.feature
